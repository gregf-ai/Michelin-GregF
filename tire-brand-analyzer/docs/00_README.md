# Documentation

This folder is intentionally small and interview-focused.

## Files

- `01_project_brief.md`: Scope, goals, and done criteria
- `02_decision_log.md`: Key architecture and product decisions
- `03_data_contract.md`: Source endpoints, schema, currency policy, and known gaps
- `04_financial_analyst_agent.md`: Financial analyst agent workflow, tools, and response policy
- `05_deployment_runbook.md`: Local, Docker, and Render deployment steps
- `06_handoff_status.md`: Current status, open risks, and immediate next tasks

## How to Use During Interview Prep

1. Read `01_project_brief.md` first to anchor the story.
2. Use `02_decision_log.md` to explain tradeoffs.
3. Use `03_data_contract.md` when asked about data quality and assumptions.
4. Use `04_financial_analyst_agent.md` when asked how the AI financial analyst works.
5. Use `05_deployment_runbook.md` for demo day readiness.
6. Update `06_handoff_status.md` at the end of each work session.

## Update Rule

Keep each file concise and operational. Prefer bullets over long prose.
